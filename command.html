The command pattern is used in the implementation of the three function buttons; clear, run and step. These button all have one instance of the ActionListener.

The text from the button is used to check which button invoked the ActionListener for the button. Since the implementation was done carefully and there are only three buttons; this allowed to do a comparision on the initial character of the button.

This pattern would prove efficient in terms of memory since only one instance of the ActionListener would be in the memory rather than 3. Even though this could be inefficient in terms of processing since string comparisons are costly. Keeping that in mind we can use the faact that the events of clicking on these buttons wont be very often and also only comparision between the first character of the string provides faster processing time.
